# Durable Memory

## User Preferences

- Wants Kxran-OS to be model-agnostic, not tied only to Codex or Claude.
- Prefers a Markdown-first local v1 before adding Git, Obsidian Sync, plugins, MCP, or automation.
- Wants token efficiency to be a first-class design goal.
- Wants analytics to combine graphs, exact metrics, and written interpretation.

## OS Principles

- The vault should help future agents avoid forgetting prior decisions.
- Session history should be indexed and summarized, not loaded wholesale.
- Weekly analytics should reveal agent learning quality, productivity output, and context efficiency.

## Durable Decisions

- V1 mode is Solopreneurs/Professionals.
- Session structure is chronological plus indexes.
- Four scoped operating `AGENTS.md` files are used as the OS spine.
