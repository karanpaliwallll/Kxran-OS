# Agent Brief

Kxran-OS is a local-first, model-agnostic Obsidian AI OS. Its purpose is to help AI agents and the user work with continuity without loading full conversation history.

## Always Remember

- The vault is the persistent state layer.
- Current project status beats old session history.
- Raw sessions are archives, not startup context.
- Keep context loads small and targeted.
- Promote durable lessons into `Context/memory.md`.
- Promote corrections into `Context/corrections.md`.

## Startup Context

For a normal session, read:

1. this file
2. `Context/retrieval-map.md`
3. relevant project `README.md` and `status.md`
4. `Sessions/active.md`

Only read older sessions when a specific historical decision, blocker, or correction is needed.

## Current Focus

Bootstrap and test Kxran-OS v1 locally before adding Git, Obsidian Sync, custom plugins, MCP, or automation hooks.
