# 07 - Sharing And Onboarding

This note explains how to share Kxran-OS with someone else.

## What To Tell Them First

Kxran-OS is a Markdown-based AI operating system inside Obsidian.

It helps by:

- keeping project truth outside chat history;
- storing durable memory;
- logging sessions;
- tracking corrections;
- reviewing weekly improvement;
- keeping AI context token-efficient.

## What They Should Read First

1. `README.md`
2. `Walkthrough/README.md`
3. `Walkthrough/01-os-map.md`
4. `Context/agent-brief.md`
5. `Projects/kxran-os/status.md`
6. `Analytics/weekly/2026-W24.md`

## How They Should Use It

For a new project:

1. Create a project folder.
2. Write a `README.md`.
3. Write a `status.md`.
4. Log sessions chronologically.
5. Promote durable lessons into memory.
6. Add corrections when mistakes happen.
7. Review analytics weekly.

## How To Explain The Token Strategy

Do not say the OS remembers by loading everything.

Say:

> The OS remembers by keeping current truth short, storing history in indexed archives, and retrieving older context only when needed.

That is the key idea.

## What They Can Change

They can change:

- folder names;
- templates;
- analytics metrics;
- project status structure;
- session note structure;
- skills and workflows;
- sync strategy.

They should preserve:

- a short always-load context file;
- current project status files;
- session indexes;
- corrections;
- a weekly review habit.

## What To Avoid

- Do not make raw session logs the main memory.
- Do not load the whole vault every session.
- Do not mix current truth with old history.
- Do not add plugins before the Markdown workflow is clear.
- Do not store secrets unless sync and backup policies are understood.
