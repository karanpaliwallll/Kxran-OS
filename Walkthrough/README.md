# Kxran-OS Walkthrough

Start here if you are new to this vault.

Kxran-OS is a local-first Obsidian AI OS. It is built from plain Markdown files so a human, Codex, Claude, Gemini, ChatGPT, Cursor, or a local model can all understand the same operating system.

The main idea is simple:

- Obsidian stores the long-term system.
- AI agents read only the context they need.
- Current project files are the source of truth.
- Session logs preserve history.
- Memory and corrections preserve lessons.
- Weekly analytics show whether the system is improving.

## What This Is Not

Kxran-OS is not a custom Obsidian plugin. It is not locked to Codex. It is not a database that forces one workflow.

It is a structured Markdown operating system that can grow into automation later.

## Fast Mental Model

Think of the vault in four layers:

1. **Current truth**: project status, context, active handoff.
2. **Memory**: durable facts, preferences, corrections.
3. **History**: session notes, daily notes, archived decisions.
4. **Reflection**: analytics and weekly reviews.

Agents should load layer 1 first, use layer 2 when needed, search layer 3 only for specific questions, and update layer 4 during review.
